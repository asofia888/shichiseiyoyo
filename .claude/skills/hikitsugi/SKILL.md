# 引き継ぎ要約

長くなった会話を仕切り直すために、新しいセッションの最初に貼る
「引き継ぎ文」を作る。以下の見出しで、事実だけを簡潔に書く。

1. 目的（この作業は何のためか・誰のためか）
2. 現在地（完了したこと。実行・確認済みの事実のみ）
3. 未完了（残タスクを優先順）
4. 決定事項（ユーザーが明示的に決めたこと。提案と混ぜない）
5. 制約・禁止事項
6. 関連ファイル（パスと一行説明）
7. 注意（ハマった点・未検証の点）

推測や提案は含めない。ユーザーの決定と、自分の提案を区別する。
出力はそのままコピーできる一つのコードブロックにまとめる。
