# 単一HTMLアプリ開発の手順書（資産二の実例）

## 前提
- 成果物は原則、依存なしの単一HTMLファイル（CSS/JS内包）。
- 外部CDN・外部サービスは、指示がない限り使わない。
- APIキーはコードに直書きせず、画面の入力欄で受け取る。

## 進め方（この順番を守る）

### 第一段階｜計画（コードはまだ書かない）
以下を箇条書きで提示し、ユーザーの「OK」を待つ。
- 主要な関数・コンポーネントの一覧（各1行説明つき）
- データの流れ（入力 → 処理 → 出力）
- 想定されるエッジケースと対処方針（空入力・異常値・API失敗・通信断）
- 仕様の曖昧な点・確認したいこと

### 第二段階｜実装
- 承認された計画のとおりに作る。
- すべての外部呼び出し・ユーザー入力にエラーハンドリングを入れる。
- コードの要所に日本語コメントを入れる。
- 仕様にない機能を足さない。良い案は「提案」として挙げるだけにする。

### 第三段階｜自己検証（報告の前に必ず行う）
- 必須の要件を一つずつ ○× で照合する。
- エッジケースごとの挙動を、実行できるものは実際に実行して確認する。
- 制約・禁止事項への違反がないか確かめる。
- 問題は指摘で終わらせず、修正まで行ってから報告する。
- 仕上げに betsujin-reviewer サブエージェントの検証を受ける。

## チェックリスト（完成の定義）
- [ ] 必須の要件がすべて ○
- [ ] 依存なしでファイル単体で動く
- [ ] スマホ表示の確認（対象なら）
- [ ] 文字化け・機種依存文字がない
- [ ] 未検証の項目が報告に明記されている
